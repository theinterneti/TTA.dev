## üöÄ Quick Start (TL;DR)

```bash

# 2. Create KB structure

# uv run python scripts/migrate-to-kb.py scan  # Already executed

# 6. Migrate Priority 1
```

## üõ†Ô∏è Tools & Scripts

### `scripts/migrate-to-kb.py`

**Commands:**

1. **`scan`** - Scan repository for documentation
   - Finds all markdown files
   - Categorizes by namespace
   - Assigns priority (1-7)
   - Saves to migration-status.json

2. **`plan`** - Show migration plan
   - Preview what will be migrated
   - Filter by priority
   - See target paths

3. **`migrate`** - Execute migration
   - Convert to Logseq format
   - Add frontmatter
   - Convert links to wiki-links
   - Move to KB
   - Create stubs in original locations
   - Support dry-run

4. **`validate`** - Validate KB structure
   - Check namespace directories
   - Count files per namespace
   - Verify structure

5. **`report`** - Generate migration report
   - Total/migrated/pending counts
   - Breakdown by priority
   - Breakdown by category

**Usage:**
```bash
# Help
uv run python scripts/migrate-to-kb.py --help
uv run python scripts/migrate-to-kb.py scan --help

# Typical workflow
uv run python scripts/migrate-to-kb.py scan
uv run python scripts/migrate-to-kb.py plan --priority 1
uv run python scripts/migrate-to-kb.py migrate --priority 1 --dry-run
uv run python scripts/migrate-to-kb.py migrate --priority 1
uv run python scripts/migrate-to-kb.py validate
uv run python scripts/migrate-to-kb.py report
```
